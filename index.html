Frequently Asked Questions

What is usectl?

usectl is a deployment platform that lets AI agents deploy your entire project with a single prompt. It includes server hosting, PostgreSQL database, S3-compatible storage, automatic SSL, custom domains, and push-to-deploy from GitHub — all for $39/month per project. It runs on K3s (Kubernetes) on Hetzner bare metal and is built by systemctl in Tbilisi, Georgia.

How do I deploy a project with usectl?

Install the CLI with curl -fsSL https://usectl.com/install.sh | bash, run usectl login to authenticate, connect your GitHub with usectl github login, then create and deploy your project with usectl projects create and usectl projects deploy. Your project will be live at an HTTPS URL in under 5 minutes. You can also paste a deploy prompt into any AI agent and let it handle everything.

How much does usectl cost?

usectl costs $39/month per project with everything included — no tiers, no usage meters, no feature gates. During the free beta, you can deploy at no cost. Founding members lock in $25/month per project forever when pricing launches.

What AI agents work with usectl?

usectl works with any AI coding agent that has terminal access, including Claude Code, Cursor, ChatGPT, Windsurf, and others. The CLI is designed for AI agent consumption with --json output on all commands. MCP connector and skill integrations for deeper agent integration are coming soon.

What's included in each project?

Every usectl project includes: container hosting with push-to-deploy from GitHub, dedicated PostgreSQL database (CloudNativePG), S3-compatible object storage (MinIO, 10GB), automatic SSL/TLS via Let's Encrypt, custom domain support, real-time monitoring and logs, and a CLI designed for both humans and AI agents.

How is usectl different from Railway or Render?

usectl is a self-hosted Railway/Render alternative that runs on dedicated Kubernetes infrastructure (K3s on Hetzner bare metal). Unlike Railway, there are no usage-based pricing surprises — $39/month covers everything including database and storage. Unlike Render, usectl is designed for AI agents first: your coding assistant can deploy, manage, and monitor your project through the CLI without you touching a dashboard. You get the same full-stack experience (hosting + database + storage + SSL) but at a flat, predictable price.

Do I need a Dockerfile?

Not always. usectl auto-detects Next.js, Node.js, and static sites and injects an appropriate Dockerfile automatically. For other frameworks like Go, Python, or Rust, you need a Dockerfile in your repo root.

Can I use my own custom domain?

Yes. Run usectl domains add --project <id> --domain yourdomain.com and add a CNAME record pointing to usectl.app. SSL certificates are provisioned automatically via Let's Encrypt and Traefik.
